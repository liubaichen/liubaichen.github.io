  1. !important （慎用）

  2. 内联样式（style）

覆盖顺序（级别高的先被覆盖）

  1. 用户代理样式表中的声明（例如，浏览器的默认样式，在没有设置其他样式时使用）。
  2. 用户样式表中的常规声明（由用户设置的自定义样式）。
  3. 作者样式表中的常规声明（这些是我们 web 开发人员设置的样式）。
  4. 作者样式表中的 !important 声明
  5. 用户样式表中的 !important 声明
  6. 用户代理样式表中的 !important 声明

级联层顺序

  1. 后加载的覆盖先加载的
  2. 设计!important的，顺序相反——先前的层中的 important 样式比后面的层以及为在层中声明的 important 样式优先级要高
  3. 内联样式比所有作者定义的样式的优先级都要高，不受级联层规则的影响

general rule

  1. 具有 important 的样式，则先前声明的层覆盖后声明的层，而不是层的优先级

  2. 其他样式，一律按照层先后顺序覆盖，层内再根据优先级判断

盒子

块级盒子 (block box) 和 内联盒子 (inline box)

一个被定义成块级的（block）盒子会表现出以下行为：

  • 盒子会在内联的方向上扩展并占据父容器在该方向上的所有可用空间，在绝大数情况下意味着盒子会和父容器一样宽
  • 每个盒子都会换行
  • width 和 height 属性可以发挥作用
  • 内边距（padding）, 外边距（margin）和 边框（border）会将其他元素从当前盒子周围“推开”

除非特殊指定，诸如标题 (<h1>等) 和段落 (<p>) 默认情况下都是块级的盒子

如果一个盒子对外显示为 inline，那么他的行为如下：

  • 盒子不会产生换行。
  • width 和 height 属性将不起作用。
  • 垂直方向的内边距、外边距以及边框会被应用但是不会把其他处于 inline 状态的盒子推开。
  • 水平方向的内边距、外边距以及边框会被应用且会把其他处于 inline 状态的盒子推开。

如何理解垂直不推开、水平推开

用做链接的 <a> 元素、 <span>、 <em> 以及 <strong> 都是默认处于 inline 状态的。

优势：灵活性/响应性

模型参数 含义
主轴 元素延伸方向
交叉轴 主轴的垂直方向

flex-direction: column/row(fault)

flex-wrap: wrap (默认情况下，所有子元素会分布在一行或一列，使用wrap可以确保子项可以自动换行，以适应容器的宽度)

flex：1 200px

强调：flex的定义是在兄弟元素css中，不是父元素

第一参数：无单位比例，表示每个 flex 项沿主轴的可用空间大小，这是一个比例，如果只设置一个数的话，无论是1还是任意其它整数都表示每个元素占用空间都是相等的；可以搭配nth-of-type()实现间隔排列

第二参数：配合wrap使用；每个 flex 项将首先给出 200px 的可用空间，然后，剩余的可用空间将根据分配的比例共享

align-items: 交叉轴方向上的分布（类似于将全部元素压缩到交叉轴上）

align-self: 对指定兄弟元素操作

justify-content: 主轴方向上的分布

order: [number]

flex项排序：

  • 所有 flex 项默认的 order 值是 0（可以比0小，也可以比0大）
  • order 值大的 flex 项比 order 值小的在显示顺序中更靠后

基于线的布置（逐个元素操作）

线布置要写在目标元素的css中，不是在父元素中

线布置可以实现元素重叠，这是下面的area布置无法实现的

很好理解，假设有1个2行、2列的grid，html中的顺序分别记为a、b、c、d，默认基于template的布置即为顺序排列。如果想让a和c交换，就把a（通过class）的grid-column和grid-row分别设为1和2，c的改为1和1；更有用的是设置为1 / 3（举例），即1到2合并（类似于word中的表格合并）

基于area的布置

适用情形：元素不多（毕竟要给全部元素命名）、自定义个性化强

比线布置更容易理解：

  1. 给每个元素命名（grid-area）
  2. 在父元素中中依次写入每个格子要填入内容的元素名称（grid-template-areas）
  • 使用.符号，让一个格子留空
  • 必须填满所有格子（否则整个grid崩溃，所有子元素挤在一个格子中）
  • 所有名字只能出现在一个连续的区域，不能在不同的位置出现

float（浮动）

传统布局方法

position（定位）

很重要，可以让网页变得花里胡哨！

显著特征：position元素不影响其它元素的布局，即position元素将出现在其他元素上方，例如创建不干扰页面上其他元素的弹窗

static定位（默认）

relative定位

absolute定位

fixed定位

sitcky定位

如果所有的父元素都没有显式地定义 position 属性，绝对定位元素会被放在元素的外面，即absolute是相对于html的坐标；在父元素中设置position: relative或absolute，可以使相对坐标调整为父元素（或者祖先元素，但不能是兄弟元素）

position可以理解为新创立独立于网页x和y轴的z轴，定义原始基准为0，则position元素位于0上方。相应地，position元素之间也有z轴上的相对关系，用z-index设置（z-index为相对值）。

fixed与absolute的区别类似于background-attachment属性：fixed是与视窗保持静止（eg 滚动网页上一直保持在固定位置的侧栏），absolute是与网页元素保持静止

sticky也非常实用，效果时允许被定位的元素表现得像相对定位一样，直到它滚动到某个阈值点（例如，从视口顶部起 10 像素）为止，此后它就变得固定了。例如，它可用于使导航栏随页面滚动直到特定点，然后粘贴在页面顶部。

小技巧：如何居中一个元素？

position按需填写，top指从50%位置放置元素，因此会因为元素本身的高导致无法居中，transform可以实现根据元素本身高度的平移，50%指元素本身高度的一半

注：背景是要有的，因为sticky元素会叠加显示

文字样式

包括字体样式和文本布局

字体样式

基本元素

浏览器的 font-size 标准设置的值为 16px，可以通过html元素修改为任意数值

  • text-transform

    : 允许你设置要转换的字体。值包括：

    • none: 防止任何转型。
    • uppercase: 将所有文本转为大写。
    • lowercase: 将所有文本转为小写。
    • capitalize: 转换所有单词让其首字母大写。
    • full-width: 将所有字形转换成全角，即固定宽度的正方形，类似于等宽字体，允许拉丁字符和亚洲语言字形（如中文，日文，韩文）对齐。

  • text-decoration

    : 设置/取消字体上的文本装饰 (你将主要使用此方法在设置链接时取消设置链接上的默认下划线。) 可用值为：

    • none: 取消已经存在的任何文本装饰。
    • underline: 文本下划线。
    • overline: 文本上划线
    • line-through: 穿过文本的线。

    你应该注意到 text-decoration 可以一次接受多个值，如果你想要同时添加多个装饰值，比如 text-decoration: underline overline.。同时注意 text-decoration 是一个缩写形式，它由 text-decoration-line, text-decoration-style 和 text-decoration-color 构成。你可以使用这些属性值的组合来创建有趣的效果，比如 text-decoration: line-through red wavy.

  • 新元素：文字阴影 text-shadow: 4px 4px 5px red;

4 个属性如下：

  1. 水平偏移，必须指定
  2. 垂直偏移，必须指定
  3. 模糊半径 - 更高的值意味着阴影分散得更广泛。如果不包含此值，则默认为 0，意味着没有模糊（相当于将文本平移，不推荐）
  4. 颜色，默认为 black

文本布局

text-align 控制文本和它所在的内容盒子对齐

  • left: 左对齐文本。
  • right: 右对齐文本。
  • center: 居中文字
  • justify: 使文本展开，改变单词之间的差距，使所有文本行的宽度相同

line-height 行高

最常用的是无单位的值，作为乘数，表示几倍行距

writing-mode: vertical-rl

两个参数: 文本方向 和 书写方向

​ vertical和horizontal指文字是垂直还是水平连接

​ rl lr (tb bt) 指文本从左到右书写还是从右往左书写(对应于水平连接,就是上到下还是下到上)

在vertical和horizontal模型下，inline和block的表现形式也发生了变化（vertical下，文字延伸方向即纵向为inline，横向为block，horizontal相反）；因此引入相对变量内联尺寸（inline-size）和块级尺寸（block-size），并由此引出一些列的逻辑值

逻辑属性，即相对参量（逻辑值表），对应于物理属性（绝对参量）。典型的有inline-size/block-size、margin-top、 padding-left和border-bottom，block-start、inline-end、block-end和inline-start（top、right、bottom、left的对应参数）

text-indent: 缩进

hyphens: auto 自动断开单词，并开启连字符

以下属性可以被hyphens替代

word-break: 指定是否能在单词内部换行

word-wrap: 指定浏览器是否可以在单词内换行以避免超出范围

样式化链接

链接状态（伪类）

  • Link (没有访问过的): 这是链接的默认状态，当它没有处在其他状态的时候，它可以使用:link 伪类来应用样式。
  • Visited: 这个链接已经被访问过了 (存在于浏览器的历史纪录), 它可以使用 :visited 伪类来应用样式。
  • Hover: 当用户的鼠标光标刚好停留在这个链接，它可以使用 :hover 伪类来应用样式。
  • Focus: 一个链接当它被选中的时候 (比如通过键盘的 Tab 移动到这个链接的时候，或者使用编程的方法来选中这个链接 HTMLElement.focus() (en-US)) 它可以使用 :focus 伪类来应用样式。
  • Active: 一个链接当它被激活的时候 (比如被点击的时候)，它可以使用 :active 伪类来应用样式。
